# 容器云 vs OpenStack
1. 容器 vs VM
  + 计算资源的高效利用
  + 运行环境管理和隔离
2. 平台提供应用的高可用
  + 应用定义好健康的标准
  + 应用崩溃或处于不健康状态，平台会自动重启应用
  + 应用不需要任何多余编码
3. 基于应用的调度和水平扩展
  + 基于资源的调度
  + 基于应用亲和性的调度
  + 基于特殊资源需要的调度

## 容器云的完整方案
1. k8s是操作系统的内核(linux kernel)
2. zcloud = k8s + GUI + 网络 + 存储 (centos, ubuntu)
3. 完全自动化，只需要一个linux集群

## 高效的k8s管理和运维
1. 隐藏k8s的细节，让用户只关注自己的应用
2. 从更高的层面展示整个系统上，应用之间，以及应用和网络存储之间的关系
3. 通过图形界面展示复杂的异步系统
4. 快速定位问题
5. 自动化的k8s升级
